# 契約管理・資金繰りシステム — 要件定義書

## 1. プロジェクト概要

請求書・契約管理がスプレッドシートで煩雑になっている問題を解決するため、BAKUSOQ・NiNKUBOXX・その他の3製品の契約管理と資金繰りを一元管理するWebアプリケーションを構築する。

### 技術スタック（推奨）
- フロントエンド: Next.js (React) + TypeScript
- データベース: Supabase (PostgreSQL) または Firebase
- UI: Tailwind CSS
- デプロイ: Vercel

---

## 2. データモデル

### 2.1 企業マスタ（companies）
BAKUSOQ・NiNKUBOXXで共通管理。同じ企業が複数製品を契約する可能性あり。

| フィールド | 型 | 説明 |
|---|---|---|
| id | UUID | 主キー |
| name | string | 会社名（必須） |
| contact | string | 担当者・連絡先 |
| note | text | メモ |
| created_at | datetime | 作成日時 |
| updated_at | datetime | 更新日時 |

### 2.2 契約（contracts）
各製品ごとに契約を管理。1企業が複数製品を契約可能。

| フィールド | 型 | 説明 |
|---|---|---|
| id | UUID | 主キー |
| company_id | UUID | 外部キー → companies |
| product_type | enum | "bakusoq" / "ninkuboxx" / "other" |
| contract_start_date | date | 契約開始日（任意の日付） |
| billing_month | string | 起算月（YYYY-MM形式） |
| billing_day | enum | 起算日 "1" or "16" |
| duration_months | integer | 契約期間（ヶ月） |
| monthly_fee | integer | 月額料金（税別） |
| monthly_close | enum | 月額の締め月 "-1"(前月) / "0"(当月) / "1"(翌月) |
| monthly_pay | enum | 月額の支払い "same_end"(当月末) / "next_end"(翌月末) / "next_10"(翌月10日) |
| has_initial_fee | boolean | 初期導入費の有無 |
| initial_fee | integer | 初期導入費（税別） |
| initial_close | enum | 初期費用の締め月（同上） |
| initial_pay | enum | 初期費用の支払い（同上） |
| has_option | boolean | オプションの有無 |
| option_name | string | オプション名 |
| option_fee | integer | オプション月額（税別） |
| option_close | enum | オプションの締め月（同上） |
| option_pay | enum | オプションの支払い（同上） |
| note | text | メモ |
| created_at | datetime | |
| updated_at | datetime | |

**契約完了日の計算**: 起算日（billing_month + billing_day）から duration_months ヶ月後の前日。
例: 起算日 2026/4/1、12ヶ月 → 完了日 2027/3/31

**入金月の計算**: 
- close offset = monthly_close の値 (-1, 0, 1)
- pay offset = pay が "next_*" なら +1、"same_end" なら 0
- 入金月 = 対象月 + close_offset + pay_offset

## 3. 画面構成

### 3.1 サイドバーナビゲーション
以下の順番で表示（折りたたみ可能）：

1. 📄 BAKUSOQ契約
2. 📄 NiNKUBOXX契約  
3. 📄 その他契約
4. ─── 区切り線 ───
5. 📊 資金繰り表
6. 🏢 企業マスタ

### 3.2 各契約ページ（BAKUSOQ / NiNKUBOXX / その他）

#### メイン画面：月別売上テーブル
- ヘッダーに「+ 契約を追加」ボタンと「📋 契約一覧」ボタン
- 月別売上テーブルがメインコンテンツ
  - 行: 各契約企業
  - 列: 月（横スクロール可能）
  - 企業名列は sticky（左固定）
  - 合計行あり
  - 入金月ベースで金額を表示（締め/払いの設定に基づくオフセット反映）
  - 初期導入費は初月のみに加算

#### サブ画面：契約一覧（「📋 契約一覧」ボタンから遷移）
- 「← 戻る」ボタンで月別売上に戻る
- テーブル: 企業名 / 契約開始日 / 起算日 / 期間 / 完了日 / 月額 / 支払条件 / 初期費用 / オプション
- 行クリックで契約編集モーダル
- 各行に削除ボタン

### 3.3 契約登録・編集モーダル

#### 契約企業選択
- 既存企業をプルダウンで選択
- 「+ 新規企業」ボタンでインライン企業登録（会社名 + 担当者を入力 → 企業マスタにも自動追加）

#### 契約期間セクション（青枠）
- 契約開始日: date picker（任意の日付）
- 起算月: month picker（契約開始日から自動提案、手動変更可）
- 起算日: プルダウン（1日 / 16日）
- 契約期間（ヶ月）: 数値入力（スピナーなし）
- 契約完了日: 自動計算表示
- 下部に「起算日: YYYY/M/D → 完了日: YYYY/M/D」表示

#### 月額料金セクション
- 月額料金（税別）: 数値入力（3桁カンマ区切り表示、スピナーなし）
- お客様お振込日: 2つのプルダウン
  - 締め: 実際の月数を表示（例: 3月 / 4月 / 5月） ← 起算月を基準に前月/当月/翌月
  - 払い: 実際の月+日を表示（例: 4月末 / 5月末 / 5月10日） ← 締め月を基準に
  - 下部に「→ 当月締め翌月末払い」のような説明テキスト表示

#### 初期導入費セクション（チェックボックスON/OFF）
- 初期導入費（税別）: 数値入力（カンマ区切り）
- 初月お客様お振込日: 月額と同じUI構成

#### オプションセクション（チェックボックスON/OFF）
- オプション名: テキスト入力
- オプション月額（税別）: 数値入力（カンマ区切り）
- 初月お客様お振込日: 月額と同じUI構成

#### メモ
- テキストエリア

### 3.4 資金繰り表ページ
- サマリーカード: 契約企業数 / BAKUSOQ契約数 / NiNKUBOXX契約数 / その他契約数
- 月別テーブル:
  - 行: BAKUSOQ / NiNKUBOXX / その他 / 売上合計
  - 列: 月（横スクロール）
  - 入金月ベースで表示

### 3.5 企業マスタページ
- 企業一覧（カード形式）
  - 会社名
  - 契約中の製品バッジ（BAKUSOQ / NiNKUBOXX）
  - 担当者・連絡先
  - クリックで編集モーダル
  - 削除ボタン
- 「+ 企業を追加」ボタン → モーダル

## 4. ビジネスロジック

### 4.1 売上計算ルール
- 各契約の月額料金は、起算月から契約期間分の月に発生
- 入金月 = 対象月 + 締めオフセット + 支払いオフセット
  - 締めオフセット: 前月=-1, 当月=0, 翌月=1
  - 支払いオフセット: 当月末=0, 翌月末=1, 翌月10日=1
- 初期導入費は起算月の1ヶ月目のみに加算（入金月はclose/payに従う）
- オプション料金は月額と同様に毎月発生（入金月はclose/payに従う）

### 4.2 日付計算
- 契約完了日 = 起算日 + 契約期間ヶ月 - 1日
  - 例: 2026/4/1起算、12ヶ月 → 2027/3/31
  - 例: 2026/4/16起算、6ヶ月 → 2026/10/15

### 4.3 起算月の自動提案
- 契約開始日の日付 ≤ 起算日 → 同月
- 契約開始日の日付 > 起算日 → 翌月
- 例: 開始日3/25、起算日=1日 → 起算月=4月

## 5. UI/UX仕様

### 5.1 金額入力
- type="number" は使わない（スピナー不要）
- inputMode="numeric" でモバイル対応
- 入力中に3桁カンマ区切り自動表示（例: 30,000）
- 数字以外の文字は自動除去

### 5.2 テーブル
- 横スクロール対応
- 先頭列（企業名/項目名）は sticky left
- 金額は右寄せ、等幅表示
- 0円の月は「—」表示

### 5.3 モーダル
- 背景クリックで閉じる
- backdrop-filter: blur
- 最大高さ88vh、スクロール対応

### 5.4 日本語フォント
- Noto Sans JP
